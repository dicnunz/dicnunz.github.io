Runnable proof that the field kit can become a validated handoff.

This is a cleanroom reference implementation: structured JSON goes through required-field validation and renders into a markdown deployment receipt with command results, decision, limits, and product feedback.

Validation

What the tool rejects

  1. Scope gaps Missing owner, task, verifier, data boundary, rollback path, human review rule, or decision owner.
  2. Receipt gaps Missing context loaded, files changed, command result, security boundary, decision, or known limits.
  3. Decision drift Any decision outside ready, repair, or reject.
  4. Feedback gaps Missing ready patterns, trust failures, or product implications.

Fit

Why this matters

The role asks for demos, reference implementations, workflow automations, technical content, product feedback, safety, and operational readiness. This reference turns the receipt pattern into runnable source.

This is not customer deployment history. It is a code-backed operating proof: a demo should produce structured evidence a second engineer can validate without the presenter in the room.
